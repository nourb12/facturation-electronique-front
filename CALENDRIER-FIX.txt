PROBLÈME IDENTIFIÉ :
  ❌ Impossible de sélectionner d'autres dates que le 30 avril 2026
  ❌ Toutes les dates étaient bloquées (considérées comme "passées")

CAUSE :
  La logique bloquait les dates passées (isPast) ET les week-ends
  Comme aujourd'hui = 30 avril 2026, toutes les dates avant étaient bloquées

═══════════════════════════════════════════════════════════════════════════════
  🔧 CORRECTIONS APPLIQUÉES
═══════════════════════════════════════════════════════════════════════════════

1️⃣ FICHIER : demo-booking.component.ts
   
   AVANT (ligne 163) :
   selectDay(day: CalendarDay): void {
     if (!day.isCurrentMonth || day.isPast || day.isWeekend) return;
     ...
   }

   APRÈS :
   selectDay(day: CalendarDay): void {
     // Allow selection of current month days that are not weekends
     // Remove the isPast check to allow booking for any future date
     if (!day.isCurrentMonth || day.isWeekend) return;
     ...
   }

   ✅ Suppression de la vérification "day.isPast"
   ✅ Permet de sélectionner toutes les dates du mois (sauf week-ends)

2️⃣ FICHIER : demo-booking.component.html
   
   AVANT (ligne 179) :
   [class.db-day--past]="day.isPast || day.isWeekend"
   [disabled]="!day.isCurrentMonth || day.isPast || day.isWeekend"

   APRÈS :
   [class.db-day--past]="day.isPast"
   [disabled]="!day.isCurrentMonth || day.isWeekend"

   ✅ Séparation visuelle : isPast reste pour le style
   ✅ Désactivation uniquement pour : autres mois + week-ends
   ✅ Les dates passées restent visuellement grisées mais cliquables
